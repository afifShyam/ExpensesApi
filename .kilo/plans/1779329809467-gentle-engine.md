# Plan: Restructure ExpenseApi to Clean Architecture-style (Single Project)

## Goal

Restructure the ExpenseApi project from its current flat layer layout into a Clean Architecture-inspired single-project layout. This organizes the code into `Domain`, `Application`, `Infrastructure`, and `Controllers` layers without splitting into multiple `.csproj` files yet.

### Identified issues to fix during restructure

1. `Error.cs` uses `System.Net.Http.StatusCode` constants (`StatusCodes.Status404NotFound`), coupling the `Domain` layer to ASP.NET Core. These must be replaced with plain integer literals.
2. `CommitmentController.cs:59` — the `Update` action calls `Deleted()` instead of `Success(value, "...updated...")`. This is a bug.
3. `ExpenseRepository.Create/Update` and `CommitmentService.Create/Update` both set `CreatedAt`/`UpdatedAt`. The architecture note says the **service** should own business timestamps, not the repository.
4. Namespace inconsistencies between original and restructured folders.

Run these bash commands to build the skeleton:


### Phase 2 — Move files with identity-preserving writes (use write tool per file)

Move each file to its new location using the `write` tool, which reads the old file, writes it to the new path, and then deletes the old file. Each move includes the namespace correction.

#### 2b. Fix `Error.cs` before moving to Domain

In `ExpenseApi/Common/Error.cs`, replace every `StatusCodes.StatusXXX` constant with the equivalent integer literal **before** writing it to `Domain/Common/Error.cs`:

```csharp
// Before
public static Error NotFound(string code, string message, string? details = null)
    => new(code, message, StatusCodes.Status404NotFound, details);

// After
public static Error NotFound(string code, string message, string? details = null)
    => new(code, message, 404, details);
```

Apply to all 5 static factory methods (NotFound→404, BadRequest→400, Unauthorized→401, Conflict→409, and the constructor call in `Validation`).

Then write it to `Domain/Common/Error.cs` with namespace `ExpenseApi.Domain.Common`.

FIX BUG in `Controllers/CommitmentController.cs:59`:

Replace:

|-------|------|----------------|
| **Domain** | `Domain/Entities/` | All entity models (was `Models/`) |
| | `Domain/Common/` | Result + Error (was `Common/`), with plain integer status codes |
| **Application** | `Application/DTOs/` | All DTOs (was `DTOs/`) |

| | `Infrastructure/Repositories/` | Repository implementations (was `Repositories/*/`) |

## Risks and mitigation

| Risk | Mitigation |
|------|-----------|
| Edit tool may forget to delete old files | Explicitly verify old files are gone after each move |
| Namespace mismatch in修补 files | Compile check (`dotnet build`) after every group of moves |
| Migration files reference old namespaces | EF Core migrations are generated; their namespaces are generated by tooling — leave as-is |
| `STATUS_CODE` replaced with `int` might not match EF Core OpenAPI generator | `int` is the same type as what `StatusCodes` references — functionally identical |
| `CommitmentController.Update` bug surfaced | Fixed inline in Phase 3 — `Deleted()` → `Success(value, "...updated...")` |
